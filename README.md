## вңҰ &nbsp; What Is This?

> **Slot Machine** is a fully animated, production-quality 2D casino game built from scratch in Unity.
> Every pixel, every bounce, every coin arc was crafted to deliver that visceral, satisfying casino feel вҖ”
> without a single trip to Las Vegas.

The game runs on a clean **component-based architecture**, powered by **DOTween** for all its animations.
ScriptableObjects keep symbols designer-friendly, while a tightly scoped game loop keeps the code maintainable and extensible.

**Start with 1000 Gold. Place your bet. Pull the handle. Win big вҖ” or try again.**

### Step 1 вҖ” Place Your Bet
Click any bet button. **Gold is instantly deducted** вҖ” coins pop from your counter, arc upward, and fly straight into the machine. You *feel* the money leaving your hands.

### Step 2 вҖ” Pull the Handle
Click the lever. It physically switches between its **idle and pulled frames**, snapping down with satisfying rigidity. The machine accepts your fate.

### Step 3 вҖ” Watch the Reels Spin
All three reels ignite simultaneously, blur streaking past. Then they stop вҖ” **one by one** вҖ” each landing with a firm mechanical bounce that shakes the whole cabinet. The stagger is deliberate. The tension is real.

### Step 4 вҖ” Win or Walk Away
Three matching symbols? **Coins erupt from the machine.** They arc through the air and slam into your gold counter вҖ” each one punching the number higher with a satisfying pop. No match? The reels reset. The handle waits.

## вҡҷпёҸ &nbsp; Core Systems

<br>

### рҹҺ° &nbsp; The Reel Engine
Three independent reels, each spinning and stopping with **staggered timing** for maximum dramatic effect. Blur sprites overlay the reel during spin, then cut away cleanly on landing. A DOTween overshoot tween delivers the mechanical bounce вҖ” configurable per symbol weight class.

### рҹӘҷ &nbsp; The Coin Economy
Two distinct animation modes power the gold flow:

| Flow | Direction | Description |
|:---|:---|:---|
| **Bet Deduction** | Counter вҶ’ Machine | Coins pop out, arc upward, vanish into the slot |
| **Win Payout** | Machine вҶ’ Counter | Coins burst outward, fly in, slam the counter with a punch animation |

Every coin is an **individual pooled object** вҖ” spawned, tweened, and recycled. The counter text counts up live as each coin arrives.

### рҹғҸ &nbsp; Symbol System
Symbols are defined as **ScriptableObjects** вҖ” `SlotSymbol` assets that designers can create, tune, and rebalance without touching a single line of code.

Each symbol exposes:
- **Sprite** вҖ” the reel face art
- **Payout Multiplier** вҖ” applied to the bet on a match
- **Is Wildcard** вҖ” wildcards substitute for any other symbol

Add a new symbol in 30 seconds. No code. No recompile.

### рҹғ‘ &nbsp; Win Detection
After all reels settle, the system reads the **center row** across all three reels, resolves wildcards, and compares values. A match triggers the payout pipeline. The entire check is a single-pass O(1) comparison вҖ” fast, clean, provably correct.

## рҹ“җ &nbsp; Architecture At a Glance

```
SlotMachineGame/
в”ңв”Җв”Җ рҹҺ°  SlotMachineController    вҶҗ Master state machine & game flow
в”ңв”Җв”Җ рҹҢҖ  ReelController           вҶҗ Spin, stop, bounce logic per reel
в”ңв”Җв”Җ рҹӘҷ  CoinAnimator             вҶҗ Spawn, arc, collect coin particles
в”ңв”Җв”Җ рҹ’°  GoldCounter              вҶҗ Balance tracking + animated text punch
в”ңв”Җв”Җ рҹ•№пёҸ  LeverController          вҶҗ Two-frame handle swap + click event
в”ңв”Җв”Җ рҹҺЁ  BlurOverlay              вҶҗ Reel spin blur sprite toggling
в””в”Җв”Җ рҹ“Ұ  SlotSymbol (SO)          вҶҗ Designer-facing symbol definition
```

The controller is the **only class that owns game state**. Every other system is a dumb, injectable component that receives commands and fires callbacks. Swapping the reel engine, coin animator, or payout formula requires touching exactly one file.
